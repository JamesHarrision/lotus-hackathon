# Project Data Integrity Audit

This report identifies all locations within the `smart-route-finder` codebase where data is currently hardcoded or simulated rather than pulled dynamically from the backend or real-time external APIs.

## 1. Super Admin Dashboard

**File:** `src/pages/superadmin/Dashboard.tsx`

*   **`systemData` Array:** The "System Efficiency Trend" area chart uses a static array of time-series data (00:00 to 23:59). In a production environment, this should be fetched from a `RoutingHistory` aggregation endpoint.
*   **KPI Evolution:** The "+12%" or similar growth indicators in the metric cards are currently static text labels.

## 2. Enterprise Admin Analytics

**File:** `src/pages/admin/Analytics.tsx`

*   **Chart Data:** Both `redirectData` (Traffic Redistribution) and `incentiveData` (Incentive Performance) are hardcoded mock arrays.
*   **Metric Totals:** The aggregate numbers in the top cards (1,402 Redirects, 94.2% Accuracy, 485 Incentives) are static constants. These should be calculated from the `RoutingHistory` and `Incentive` tables in the database.

## 3. Customer Perspective (Branch Telemetry)

**File:** `src/pages/EnterpriseDetails.tsx`

*   **`useDemoSimulation(true)`:** While the branches and their initial loads are fetched from the database, the "live movement" of the progress bars is generated by a local hook that adds random jitter to the numbers. This is for demonstration purposes to show the "Wow" factor of real-time reacting UI.
*   **`estTime` & `dist` Calculation:** The distance is calculated using the Haversine formula (as the crow flies), and the ETA is a simple mathematical estimation (Dist / Constant Speed). A real production app would use the OpenRouteService API (already integrated in the Admin Branch Editor) to get real road-network distances and traffic-aware ETAs.

## 4. Landing Page

**File:** `src/pages/Landing.tsx`

*   **Testimonials/Stats:** All marketing stats (e.g., "50% Reduction in wait times") and user quotes are static text content designed for high-conversion messaging.

## 5. Branch Incentives

**Status:** Placeholder
Throughout the UI, "Incentives" are displayed as a core feature, but the underlying logic for *issuing* a unique coupon code and *redeeming* it at a physical location is currently a UI representation. The database schema supports it, but the full lifecycle is yet to be fully wired to the backend routing triggers.

---

> [!NOTE]
> All other systems, including **User Authentication**, **Enterprise Management**, **Branch CRUD**, and **Routing History Logs**, are fully dynamic and connected to the SQLite/Prisma backend.
